// Influential Projects

While I completed dozens of cool projects at NAU, the following three contributed the most to my learning, and were also the coolest. They are not in any particular order.

1. pEEp Web Portal

For our senior capstone, our team was tasked with making an interactive web portal for a smart bird feeder. Over the course of two semesters, we went through the entire requirements acqusition process, iterated through alpha and beta prototypes, and completed extensive documentation.

Our end product was a full-fledged web application (based on Flask) that included a public community forum, profile systems, and image storage and management utilities. This project taught me how powerful a web framework like Flask could be, and opened my eyes to my potential as a backend or full stack developer. What the UI lacks in luster, our clean backend more than makes up for.

2. Multiprocessing Operating System Simulator

This was one of the largest and most involved projects I worked on during my undergrad. Throughout the semester, we worked to create a Multiprocessing Operating System Simulator. After four phases that each built on eachother, I had thousands of lines of code split between a dozen seperate libraries and a simulator that ran processes according to a parameterizable scheduling scheme. The first phase had two main objectives: first, we had to code a parser that could scan configuration files and set simulator parameters, and second, we had to save the program files into meaningful data structures for later use. In the second phase, we implemented the "first-come, first-serve" and "shortest job first" scheduling schemes to batch process the program files - this was the first time our sim actually worked as a crude OS. During the third phase, we implemented virtual memory management and ensured that each process could be allotted sufficient memory, and also ensured each process subsequently relinquished that memory back to the OS after it had finished. In the fourth and final phase, we implemented preemptive scheduling for the schemes that we had coded during phase 2 ( "shortest job first" and "first-come first-serve"), while also providing support for "round robin" scheduling. In the end, we were left with a simulator that could run a set of processes according to 5 different scheduling schemes, half of which used POSIX threads and simulated interrupts for multiprocessing.

3. Recursive Ray Tracer

This was by far my favorite project of all time, because, for the first time, it really felt like our code was coming to life. We started with a simple raycaster that supported planes and spheres, but then extended this into a full blown raytracer by implementing lighting, reflectivity, and textures. As you can see, we definitely had some fun with our test files. This entire project was done in C, which forced us to handle all the raw pixel manipulation on our own and taught us a significant amount about low-level computer graphics.
